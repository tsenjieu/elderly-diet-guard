# 【舒酸與三高守護者】高齡慢性病飲食管理 LINE 自動化助手企畫書

## 一、 專案背景與痛點分析
隨著高齡化社會來臨，許多長輩同時深受「痛風、高血壓、糖尿病、高血脂」等慢性病（痛風與三高）所苦。臨床觀察發現，長輩在日常飲食面臨以下困境：
*   **決策焦慮：** 知道要控制飲食，但面對食材時常混淆（例如：木瓜對痛風好，但糖尿病可以吃嗎？）。
*   **資訊過載：** 現有的衛教單張字體過小、紙本易丟失，且無法即時查詢。
*   **數位鴻溝：** 市售健康 App 介面過於複雜，長輩操作意願低落。

## 二、 專案目標
本專案旨在利用 LINE 官方帳號打造一個**「極簡化、直覺式」**的飲食決策工具，目標達成：
1.  **即時輔助：** 透過「三色燈號」在 3 秒內判定食材是否適合多病發作者。
2.  **健康賦能：** 降低長輩對家屬或醫護的詢問依賴，提升自主管理自信。
3.  **預防發作：** 透過正確的營養攝取與水分監測，降低急性痛風發作率及穩定血壓、血糖與血脂。

## 三、 核心邏輯：個人化飲食紅綠燈
為避免多重慢性病的飲食原則產生衝突（例如：糖尿病適合的高纖食材，可能屬於痛風的高普林禁忌），本系統採用**「個人化勾選機制」**：
1.  **初次綁定：** 進入 LINE 帳號時，由長輩或家屬勾選目前患有的疾病（痛風、高血壓、糖尿病、高血脂）。
2.  **動態燈號判定：** 系統根據使用者的疾病組合，動態套用適用的「低普林、低鈉、低糖、低飽和脂肪、高纖」原則，判定屬於以下三類：

### 🔴 第一類：絕對禁止 (紅燈區) —— 避免三病惡化
*   **關鍵字：** 酒精、高果糖、加工、濃縮。
*   **食材：** 啤酒、手搖飲、內臟類、濃縮肉湯（火鍋底）、醃製醬菜、培根香腸。
*   **原因：** 快速提升尿酸、推升血糖，且過多的鈉鹽、飽和脂肪會加劇高血壓與高血脂。

### 🟡 第二類：限制使用 (黃燈區) —— 需計量攝取
*   **關鍵字：** 精緻澱粉、高甜水果、飽和脂肪。
*   **食材：** 白飯、白麵條、紅肉（牛羊豬）、荔枝、釋迦、鳳梨、乾貨（乾香菇）。
*   **做法：** 每餐控制在「一掌心」份量，澱粉以半碗為限。

### 🟢 第三類：最適合吃 (綠燈區) —— 三病守護神
*   **關鍵字：** 高纖、鹼性、低脂、高維生素 C。
*   **食材：** **芭樂、木瓜**、黑木耳、茄子、低脂無糖優格、豆腐。
*   **原因：** 芭樂維生素 C 助排尿酸；茄子與木耳助代謝且對血管友善；低脂乳製品有助降尿酸且符合降血壓(DASH)飲食。

## 四、 功能設計與介面 (LINE 官方帳號架構)

### 1. 介面設計 (UX/UI)：長輩友善模式
*   **圖文選單 (Rich Menu)：** 底部固定顯示三個大尺寸彩色按鈕（紅、黃、綠）。
*   **大字體圖卡：** 所有回覆訊息均採用大字級（24pt 以上），並搭配食物真實照片。

### 2. 智慧互動功能
*   **語音搜尋系統：** 長輩按下 LINE 語音鍵說「我今天想吃芭樂」，系統自動辨識並回傳：
    > 「🟢 **放心吃！** 芭樂糖分低、維生素 C 多，對您的尿酸和血糖都很好喔！」
*   **關鍵字自動回覆：** 建立後台資料庫，涵蓋**台灣傳統市場常見 100 種食材**，確保貼近長輩日常採買習慣，並給予多病整合後的專業建議。

### 3. 每日飲食紀錄功能 (長輩友善極簡版)
*   **純查詢與紀錄雙軌並行：** 
    *   **僅查詢確認：** 長輩單純用語音或文字詢問時，系統僅回覆健康燈號與專業建議，不強制紀錄，滿足「隨時確認、而非要吃」的需求。
    *   **一鍵式快速紀錄：** 在回覆建議下方，會貼心附帶 `[ ➕ 紀錄到今日午餐 ]` 等直覺按鈕。若長輩確實準備享用，點擊即記錄；若只是查詢，則略過按鈕即可。
*   **極簡紀錄欄位設計：**
    *   **時間時段：** 早餐、午餐、晚餐、點心。
    *   **食物名稱：** 如芭樂、白飯（由查詢結果自動帶入）。
    *   **健康燈號：** 紅、黃、綠（系統自動判定，方便家屬一眼掌握）。
    *   **預估份量：** 一碗、一掌心、一小盤（直覺描述）。

### 4. 主動健康提醒與家屬連動 (Push & Connectivity)
*   **飲水進度條：** 每日早上 10:00、下午 15:00 推播：「請喝 300cc 的水，幫您把尿酸排出去！」
*   **溫馨小語：** 結合氣功 (Qigong) 概念，提醒「飯後散步練氣、幫助血糖平穩」。
*   **子女遠端關懷：** 家屬可透過 LINE 連結查看長輩的「飲食紅綠燈圓餅圖」（如：今日綠燈 80%、紅燈 10%），掌握健康狀況，並可發送語音或貼圖鼓勵。

## 五、 技術實作與階段規畫
1.  **資料準備期 (Week 1-2)：** 整理整合型飲食資料庫，定義每種食材對痛風、血壓、血糖、血脂的影響權重。
2.  **開發測試期 (Week 3-4)：** 使用 Python 串接 LINE Messaging API，設定自動回覆邏輯與圖文選單。
3.  **封閉試用期 (Week 5-6)：** 邀請長輩及其家屬參與試用，收集操作反饋。
4.  **正式上線 (Week 7)：** 提供專屬二維碼 (QR Code) 讓長輩掃描加入，並印製「冰箱貼紙」作為實體輔助。

## 六、 預期效益
*   **對於患者：** 降低飲食錯誤導致的急診次數，提升生活品質。
*   **對於家屬：** 減輕長期照護中的監督壓力，不需反覆叮嚀。
*   **對於社會：** 透過數位工具縮減健康不平等，實現高齡數位賦能。

## 七、 結語
本計畫不只是軟體開發，更是一場「餐桌上的數位革命」。透過將專業的醫學指引轉化為直覺的顏色信號，我們能讓長輩在多重慢性病的束縛中，重新找回「吃得安心、過得健康」的主導權。
